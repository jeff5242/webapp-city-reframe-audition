# 待確認清單 — 法規適用與審查流程

**用途：** 向都更學會專家、承辦人員、客戶確認 domain 專業問題，作為系統審查邏輯與產出格式的依據。
**整理日期：** 2026-07-06（依副總會議回饋）

> 每題標明「問誰」與「為什麼要問（影響系統哪一塊）」，方便中間人轉達與回覆。

---

## A. 給都更學會 / 法規專家

### A1. 法規版本適用原則（最優先）
**問題：** 都更審查的適用法規版本，是否**僅以申請書送核日／報核日**為唯一基準？
**延伸：** 標的物本身的條件（如建物年份、是否位於土壤液化潛勢區、既有違建）是否會**改變適用法規版本或審查標準**？
**為什麼問：** 系統目前以報核日對應當年法規（107/108/111/113 年版）。若標的物條件會改變適用，需另加標記待人工判斷。
**我方初步立場（供專家驗證）：** 代表政府審都更時，違建就是違建、法律條件一致，建商的合法/違法補償是建商的事，多半不影響政府端的容積審查，故**送核日期應為正確基準**——但需專家確認。

### A2. 「關鍵數字」的法定完整清單
**問題：** 審議資料表中，**哪些數字**是審查時一定要前後一致性比對的？（例：申請容積率、基準容積、獎勵樓地板面積、法定/實設停車位、無障礙車位、充電車位、更新單元面積、同意比率、地號、有無公有地…）
**為什麼問：** UI 要把「關鍵數字」改成看得懂的明確清單，需先確定法定應檢查的完整項目。

### A3. 未來報告書類型的法規範圍
**問題：** 除了事業計畫書、權利變換計畫書，未來還要涵蓋哪些類型（事業概要、危老、其他）？各自適用哪些法規？
**為什麼問：** 界定法規知識庫的擴充範圍（副總 #8 兩階段架構、危老擴充）。

---

## B. 給承辦人員 / 中間人

### B1. 審查流程六大步驟
**問題：** 承辦人員實際的「審查流程」有哪幾個步驟？每步驟產出什麼？
**為什麼問：** 系統產出要對準承辦的實際流程（副總 #6：從審核意見到協助產出審核報告）。

### B2. 審核意見書格式
**問題：** 承辦往上級機關送的**意見書格式**長什麼樣？哪些欄位可由系統自動填、哪些需人工填寫（藝術性/建議文字）？
**為什麼問：** 第一階段要把審核意見**結構化輸出**成可貼進意見書格式，加速人工彙整（副總 #6）。

---

## C. 給客戶

### C1. 手寫欄位實際比例
**問題：** 實際申請文件中，**手寫欄位**（如填表日期）的比例大約多少？多為打字/簽名，還是常有整段手寫？
**為什麼問：** 決定是否投入手寫辨識（副總 #3：手寫另列進階項目，先確認佔比再決定）。

---

## 回覆後對應的系統動作

| 待確認 | 回覆後系統要做 |
|--------|--------------|
| A1 法規適用原則 | 文件明列適用原則；標的物例外加「待人工判斷」標記 |
| A2 關鍵數字清單 | UI 把「關鍵數字」改成明確項目清單 + 來源頁 + 比對結果 |
| A3 報告書類型 | 規劃法規知識庫擴充範圍 |
| B1/B2 流程與格式 | 審核意見結構化輸出，對準意見書格式 |
| C1 手寫比例 | 決定手寫辨識是否列入開發 |
