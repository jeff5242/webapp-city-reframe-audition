# 會話摘要（2026-07-07 ~ 07-11）

臺北市都市更新審議自動審查 webapp。**主權/地端模式**（無 API key、Track B LLM 關閉、
純 Track A 規則 + 地端 PaddleOCR）。以下整理這幾天的成果、錯誤與教訓。

---

## 一、完成的主要工作

### 副總回饋 10 項 — 全數完成
1. 計算三段式（申請填報／依法規應為／系統核算）
2. 關鍵數字清單（值 + 來源頁 + 定位）
3. OCR 審議資料表 bbox 表格重建（+ 多輪修正，見下）
4. 分級重命名（必修→必檢）+ 圖例獨立卡片
5. highlight 定位：問題頁紅框截圖內嵌報告 + PDF inline 開啟（免下載）
6. 審核意見結構化輸出（可複製）
7. 版本適用原則（文件）
8. **兩階段 AI 架構圖**（Artifact）
9. **KPI 效能對比**（Artifact + `auditor/eval/benchmark.py` 基準工具）
10. golden-case demo 驗證

### 用戶複驗回饋修正（大魯閣／合家歡）
- 逗號千分位還原（1,406 被讀成 406）、審查時間改台北時區 UTC+8
- 停車位用字全站對齊：法定(含無障礙)汽車停車位、實設汽車停車位
- 實設汽車停車位 = 平面+機械+無障礙+充電（大魯閣 73）+ 分項明細
- 報核日期分列（事業計畫 / 權利變換各自審議資料表報核日）
- 目錄誤標修正（申請書標到目錄頁 → from_toc 旗標 + 內容頁優先）
- 送審類別以標籤為錨判讀（文字值 + 勾選框）

### OCR 強化 — 診斷、實測、路線
- **`/debug/ocr` 診斷端點**：回傳指定頁原始 OCR 偵測，用來查「PaddleOCR 實際怎麼切 token」，不再盲猜。
- **zoom5 修正**：審議資料表 bbox 重建改 zoom 5（zoom 3 會掉字）。實測大魯閣：容積獎勵 1406、
  法定 46、實設 71、無障礙 2、送審類別 C。
- **Docling+RapidOCR 評估**：能救回數字、中文標籤乾淨，但 zoom5 已補大半、且拉進 torch（映像 +2~3GB）、
  每頁 ~52s。結論：輕量補丁優先，VLM 留待第二階段 GPU。
- **模板錨定 Track-A tier（B）**：審議資料表固定版面，裁固定欄位區單獨 OCR 補基準容積 +
  衍生容積獎勵上限（=基準×50%）→ CALC-001 可核。**對齊驗證**（模板抓的獎勵樓地板需與已知值相符）
  確保只對標準版面生效、不誤填。

### 第二階段訓練工具鏈（`審議資料表_訓練標註/`）
- `docs/審議資料表_OCR訓練規劃.md` — 模板錨定 + VLM 微調兩軌、資料規格、參考專案（LLaMA-Factory、
  Qwen2.5-VL、olmOCR、PPOCRLabel…）。
- 14 張訓練圖標註骨架（manifest + 欄位 JSON + OCR 半自動預填）。
- **標註工具**（`/label` 上線）：單一自足 HTML，看圖+改欄位+提交到伺服器（S3）。承辦 `GET /label/submissions` 收回。
- `make_annotations.py`、`make_label_tool.py`、`reexport_highres.py`（高解析重輸出）、
  `vlm_zeroshot_poc.py`（Qwen2.5-VL 零樣本）、`template_extract_poc.py`。

### 維運
- **EC2 磁碟清理**：`/home/www` 85%→59%（清出 24GB）。根因 Docker Root Dir 在 /home/www，
  build cache 堆到 19.86GB。用 `docker builder prune -a -f` + `image prune -a -f`（安全）。
- 部署精簡：exclude 大型本機 PDF/zip/圖，上傳 6671→3721 檔、800MB→2.7MB。

## 二、錯誤與教訓（踩過的雷）

| 錯誤 | 根因 | 教訓 |
|---|---|---|
| 放行 IP 仍 SSH timeout | 沙箱 Bash egress IP ≠ berth 真實 SSH 來源 IP | 取 IP 一律 `dangerouslyDisableSandbox=true`、雙來源交叉驗證 |
| 反覆部署失敗 | 家用浮動 IP 每幾分鐘跳 | 放行後**立刻**部署；demo 期間可暫開 `1.160.0.0/16` |
| 用戶看到舊值以為沒修 | 報告是產生時套用；舊報告=修正前 | 改完要**重新上傳跑一次**才看得到；用 `/debug/ocr`+真實資料自我驗證 |
| 逗號值 406、基準容積抓不到 | zoom3 OCR 掉字；標籤 OCR 沒讀到 | zoom5；模板錨定裁區補值 |
| RapidOCR 接幾何拼表全 None | 幾何拼表綁死 prod PaddleOCR 的斷詞 | 換引擎要重調 → VLM（image→JSON）才是穩健解 |
| 停車位重複改仍不對 | 只改了 key_numbers、沒改明細表/rule 訊息/method_docs | 用字要**全站掃**（grep 面向用戶字串） |
| numpy 2.0 弄壞 Docling | 裝 onnxruntime 升級 numpy | paddle/docling 需 numpy<2；裝完要檢查 |
| 磁碟 85% | Docker Root Dir 在 /home/www、build cache 19.86GB | 定期 `docker builder prune`；部署 exclude 大檔 |
