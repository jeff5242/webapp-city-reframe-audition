# 會話摘要（2026-07-11 ~ 07-12）

臺北市都市更新審議自動審查 webapp。這次橫跨：OCR 資料整備 → PII 事故清除 →
PP-OCRv5 升級（把共用主機弄掛）→ **VLM 路線驗證（Qwen2.5-VL 零樣本 ~93%，決定性成果）**。

## 一、成果 / 里程碑

### 🏆 VLM 路線驗證成功（本次最大成果）
在 ixCSP 租的 **RTX-5090** 上跑 **Qwen2.5-VL-7B**：
- **零樣本（不訓練）欄位準確率 82%（strict）→ 88%（新 prompt + 寬鬆比對）→ 排除壞 gold 的文山 = ~93%**。
- 涵蓋**全 21 欄**（案名、地號、實施者、日期…），舊 PaddleOCR+幾何拼欄只能做數值那幾欄、還把主機弄掛。
- **結論：VLM（圖→JSON）完勝 OCR+拼欄。瓶頸已從「模型」變成「gold 品質」。**
- 詳見記憶 [[project-vlm-beats-ocr-pipeline]]。

### OCR 資料整備
- 13 份原始 PDF 分類/統一命名（`案名_文件類型_版本.pdf`）/去重（SHA-256 抓到 1 份重複）；靠讀 PDF 內案名確認配對（11401=康寧段291-2、1130902=新潤/士林芝山166）。
- 14 張審議資料表頁 **300 DPI 重輸出**（冠德審議表跨 3 頁 p13-15，做拼接圖）。
- 客戶用 `/label` 完成 **14/14 人工標註 gold**（已推 server S3 + 本機備份）。

### PP-OCRv5 + OpenCC（分支 feat/ppocrv5-opencc）
- `ocr_reader.py` 從 paddleocr 2.x `chinese_cht` → PP-OCRv5（3.x predict API）+ OpenCC 簡→繁 + `_infer()` 集中。
- 實測 vs chinese_cht：數值 5/9→9/9。**但部署時把共用主機弄掛（見下）。**

## 二、錯誤與教訓（踩過的雷）

| 事故 | 根因 | 教訓 / 記憶 |
|---|---|---|
| **公開 repo 提交了含 PII 的訓練標註** | 訓練 JSON/標註工具 HTML(內嵌影像)含地號/姓名 | filter-repo 清全歷史 + force-push；八進位路徑對不上、工作區重置沖掉 gitignore、所有分支都要 force-push。見 [[feedback-pii-never-commit-training-data]] |
| **PP-OCRv5 部署把 4GB 共用主機弄掛（load 484）** | 重 ML 塞 4GB 共用機 + 無容器記憶體上限 → swap 到 gp2 → I/O 死鎖 | 容器必設 `--memory`；`swappiness=1`；gp2→gp3；重 ML 要 resize/GPU 機。見 [[feedback-deployment-lessons]] |
| paddle 3.0 CPU 撞 oneDNN op bug | mkldnn 後端在 CPU 有 bug | **`enable_mkldnn=False` 建構參數**才有效（env FLAGS 對新版 PIR executor 無效） |
| PP-OCRv6 medium zoom5 推論 >100s 逾時 | 4960px 太大 + medium 模型重 | OCR 前 `_cap_size` 壓到 2000px（reconstruct 列分群是相對幾何，縮圖安全） |
| VLM LoRA 訓練 OOM（24GB） | 4960px 圖 → 視覺 token 爆 | `image_max_pixels` 降圖 + `gradient_checkpointing`；再不行 `quantization_bit: 4`(QLoRA) |
| LLaMA-Factory 裝完 torchaudio ABI 不合 | pip 拉的 torchaudio 與 NGC torch 不相容 | 只做影像→`pip uninstall torchaudio` + 把 mm_plugin 的 import 包 try/except |
| ixCSP Pod 卡「等待中」 | 選了獨佔 2×5090「庫存有限」，排不進節點 | 看事件(FailedScheduling)；改**共享單張 24GB**就排得進；7B LoRA 24GB 夠 |

## 三、GPU 訓練工作流（ixCSP 5090，可重複）
見記憶 [[reference-gpu-vlm-training-workflow]]。要點：
- 資源選**共享 24GB 單卡**（獨佔常缺貨）；鏡像 **pytorch:25.10-py3**（NGC，支援 5090/Blackwell cu128）；開共享記憶體；掛 50GB volume（模型快取 + LoRA 輸出持久化）。
- 傳檔（Pod 只有終端機、無檔案總管）：**ngrok 臨時通道 + wget**（含 PII 用亂數路徑、抓完即關）。
- 訓練包生成器：`審議資料表_訓練標註/build_bundle.py`（gold+圖 → LLaMA-Factory sharegpt 多模態）。
- LoRA 產出可攜（下載回本機/on-prem GPU），**推論需 GPU（≥16GB，量化≥8GB），不能在 4GB 主機跑**；live 辨識因 PII 要放 **on-prem GPU**。
