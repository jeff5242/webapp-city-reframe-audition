本系統執行的審查項目

文件完整性

DOC-001
申請書存在
確認封面是否附有申請書
DOC-002
切結書存在
確認封面是否附有切結書
DOC-003
委託書至少 1 份
識別委託書份數及委託用途
DOC-004
審議資料表存在
定位並確認審議資料表頁面

審議資料表欄位核對

FORM-001
送審類別已勾選
A-1 / B-1 / B-2 / C / D
FORM-002
填表日期已填寫
審議資料表填表日期欄位
FORM-003
充電車位欄位存在
111年版新增欄位，偵測是否使用舊版格式

公式計算驗算

CALC-001
容積獎勵不超過上限 必修
合計獎勵樓地板面積 ≤ 容積獎勵上限（都更條例第65條）
CALC-002
無障礙停車位精確計算 必修
依建築技術規則第167條之六計算應設輛數：≤50輛→1輛，每增加50輛再加1輛
CALC-003
實設停車位不低於法定要求 必修
實設汽車停車位 ≥ 法定汽車停車位
CALC-004
容積獎勵上限計算驗算
確認表中上限 ≈ 基準容積 × 50%（都更條例第65條）

個資保護與用詞核對

PII-001
高風險個資偵測
掃描電話、住宅地址（含弄號）、身分證字號
TERM-001
法定用詞錯誤
偵測常見錯誤用詞，如「協議合件」→「協議合建」、「計劃書」→「計畫書」
CONS-001
關鍵數字前後一致性
比對審議資料表與主文各章節的停車位數量、容積獎勵面積是否吻合

POC 功能邊界說明

本版本已涵蓋（純規則，零 API 費用）

  • ✅ 文件完整性（4 條）
  • ✅ 審議資料表欄位核對（3 條）
  • ✅ 停車位與容積數值計算驗算（4 條）
  • ✅ 個資風險掃描（Regex）
  • ✅ 法定用詞錯誤清單比對（15 個詞）
  • ✅ 審議資料表 vs 主文數字一致性

需要 LLM 才能擴充的項目（未來版本）

  • ⬜ 各章節敘述邏輯一致性
  • ⬜ 容積獎勵各項加總自動核算
  • ⬜ 公聽會記錄完整性
  • ⬜ 權利變換試算比例驗算
  • ⬜ 掃描式 PDF（需 OCR）
