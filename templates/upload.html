本系統執行的審查項目

文件完整性

DOC-001
申請書存在
確認封面是否附有申請書
DOC-002
切結書存在
確認封面是否附有切結書
DOC-003
委託書至少 1 份
識別委託書份數及委託用途
DOC-004
審議資料表存在
定位並確認審議資料表頁面

審議資料表欄位核對

FORM-001
送審類別已勾選
A-1 / B-1 / B-2 / C / D
FORM-002
填表日期已填寫
審議資料表填表日期欄位
FORM-003
充電車位欄位存在
111年版新增欄位，偵測是否使用舊版格式

公式計算驗算

CALC-001
容積獎勵不超過上限 必修
合計獎勵樓地板面積 ≤ 容積獎勵上限（都更條例第65條）
CALC-002
無障礙停車位精確計算 必修
依建築技術規則第167條之六：≤50輛→1輛，每增50輛加1輛
CALC-003
實設停車位不低於法定要求 必修
實設汽車停車位 ≥ 法定汽車停車位
CALC-004
容積獎勵上限計算驗算
確認表中上限 ≈ 基準容積 × 50%（都更條例第65條）

個資保護與用詞核對

PII-001
高風險個資偵測
掃描電話、住宅地址（含弄號）、身分證字號
TERM-001
法定用詞錯誤
偵測常見錯誤用詞，如「協議合件」→「協議合建」
CONS-001
關鍵數字前後一致性
比對審議資料表與主文各章節的停車位數量、容積獎勵面積

POC 功能邊界說明

✅ 本版本已涵蓋 — 規則引擎（零 API 費用）

  • • 文件完整性（4 條）
  • • 審議資料表欄位核對（3 條）
  • • 停車位與容積數值計算驗算（4 條）
  • • 個資風險掃描（Regex）
  • • 法定用詞錯誤清單比對（15 個詞）
  • • 審議資料表 vs 主文數字一致性
  • • 報核日期自動選擇審查法規版本（107/108/111/113 年版）
  • • 兩份文件（事業計畫 + 權利變換）分開上傳
  • • 版次變更差異比對（SQLite 歷史記錄）
  • • 標註 PDF 下載（PyMuPDF 標記問題頁）
  • • 檔案 AES-256 加密儲存於 S3，解密僅限授權伺服器
  • • 法規知識庫 Wiki（107/108/111/113 年版對照，逐年修訂 diff）

✅ 本版本已涵蓋 — AI 語意審查（需 ANTHROPIC_API_KEY）

  • • 文字頁 / 掃描頁自動分流（Docling 結構解析 + Surya OCR）
  • • LLM 全文語意掃描（Claude Haiku）：錯別字、語意矛盾、法規違反
  • • 結構欄位自動抽取與驗證：容積獎勵上限（≤ 40%）、同意比率、報核日期一致性
  • • 高風險發現 Sonnet 二次驗證
  • • 同意比率門檻依版本切換（111年：≥ 2/3 / 3/4；113年：≥ 4/5 / 4/5）
  • • AI 發現與規則引擎發現合併顯示於同一份報告
