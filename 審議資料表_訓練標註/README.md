# 審議資料表 訓練標註骨架

把客戶提供的 14 張審議資料表 PNG 建成**可訓練的標註資料**：每張一份欄位 JSON，
用 OCR 半自動預填，標註者只要**對照原圖「改錯／補漏」**，不用從零填。

## 目錄內容
- `manifest.json` — 14 張樣本索引（案名／文件類型／版本／自動填欄數）。
- `NN_<檔名>.json` — 每張的標註檔（見下方格式）。

## 標註檔格式
```jsonc
{
  "image": "單頁/大魯閣-事業計畫….png",   // 對應圖檔
  "meta": { "case": "大魯閣", "doc_type": "事業計畫", "version": null },
  "fields": {                              // ← 標註者主要工作區
    "基準容積": null,                       // OCR 預填了就有值；null = 待人工填
    "合計獎勵樓地板面積": null,
    "法定汽車停車位(含無障礙)": null,
    "實設汽車停車位_平面": 26,              // 已自動填（仍請核對）
    "實設汽車停車位_機械": 45,
    "送審類別": null,
    "填表日期": null, "報核日期": null, ...
  },
  "_note": "請對照原圖檢查/修正 fields；null 表 OCR 未自動填、需人工填。",
  "_ocr_text_reference": ["…OCR 讀到的文字…"]  // 對照用；數字通常準、中文標籤可能有錯字
}
```

## 標註規則（重要）
1. **只標「值」**，不用標框（VLM image→JSON 訓練不需 bbox）。
2. **讀不到／表上空白 → 填 `null`**（例如充電「–」→ null）。教模型「不確定就回 null」，
   別亂編（審查最怕假數字）。
3. **數字照表填**：含小數與千分位（如 `2812.00`；停車位為整數）。停車位「無障礙／充電」
   請填**實設**那一格的值。
4. **送審類別**：填代碼或敘述皆可（B-1 / 審議會版…），以表上勾選或填寫者為準。
5. `_ocr_text_reference` 只是輔助——**數字通常準、中文標籤常有錯字**（低解析 OCR），
   一律以**原圖**為準。

## 欄位清單（schema）
案名、送審類別、基地地號、更新單元面積、戶數、法定建蔽率、法定容積率、
基準容積、合計獎勵樓地板面積、獎勵比率、法定汽車停車位(含無障礙)、
實設汽車停車位_平面、_機械、_無障礙、_充電、法定機車停車位、實設機車停車位、
填表日期、報核日期、實施者、評價基準日。

## ⚠️ 提升自動預填的關鍵：解析度
目前 PNG 多為 ~1400×1000（≈120 DPI），OCR 對**中文標籤**辨識差、自動填只成功 0–3 欄。
**請從原始 PDF 以 ≥250 DPI 重新輸出**（約 3000px 寬）再重跑預填，自動填率會大幅提高，
標註更省工。

## 產出流程（可重跑）
`scratchpad/make_annotations.py`：走訪 PNG → RapidOCR → reconstruct 預填 + 文字參照 →
輸出本目錄。換高解析圖或加新樣本後重跑即可。
